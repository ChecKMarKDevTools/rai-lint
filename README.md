## ü§ñ What is this?

CheckMarK RAI Lint enforces **Responsible AI (RAI) attribution** in every commit. No more "who wrote this?" moments. No more mystery code. Just honest, trackable AI contributions.

```mermaid
%%{init: {'theme':'dark'}}%%
flowchart LR
    A[Developer Commits] --> B{Has AI Footer?}
    B -->|Yes| C[Commit Accepted ‚úÖ]
    B -->|No| D[Commit Rejected ‚ùå]
    C --> E[Clear AI Attribution]
    D --> F[Add Footer & Retry]
```

### Why does this exist?

Because transparency matters. When AI writes code, everyone should know. This isn't about fear or compliance theater‚Äîit's about building trust and maintaining clear audit trails.

## üì¶ Installation

### Node.js / Commitlint

```bash
npm install --save-dev @checkmarkdevtools/commitlint-plugin-rai
```

**Configure in `commitlint.config.js`:**

```javascript
export default {
  extends: ['@commitlint/config-conventional'],
  plugins: ['@checkmarkdevtools/commitlint-plugin-rai'],
  rules: {
    'ai-attribution-exists': [2, 'always'],
  },
};
```

### Python / Gitlint

```bash
uv add checkmarkdevtools-gitlint-plugin-rai
```

**Configure in `.gitlint`:**

```ini
[general]
contrib = checkmark_rai_lint.rules.RaiFooterExists
```

```yaml
pre-push:

    sonar-scan:
      run: npx dotenv -- npx sonar-scanner
```

<details>

```bash
npx husky add .husky/commit-msg 'npx commitlint --edit $1'
```

</details>

<details>
<summary><b>pre-commit</b></summary>

```yaml

  - repo: local
    hooks:
      - id: gitlint
        name: gitlint
        entry: gitlint
        args: [--msg-filename]
        language: python
        stages: [commit-msg]
```
